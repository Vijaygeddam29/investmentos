Backend API Design (Small but Important Change)

Your endpoint:

GET /api/portfolio/builder

is correct, but add two optional parameters now so you don’t need a breaking change later.

Recommended API:

GET /api/portfolio/builder

Query params:

strategy=rocket|fortress|wave
size=10
weightMethod=equal|score|risk
sectorCap=2
country=all|usa|uk|india
marketCap=all|large|mid|small

Why?

This allows the Portfolio Builder to act like a mini institutional portfolio constructor.

Example request:

/api/portfolio/builder?strategy=rocket&size=15&weightMethod=risk&country=usa

3️⃣ Sector Cap Algorithm (Needs One Extra Step)

Your current logic:

max 2 stocks per sector

But if the top 10 list has many tech companies, you could end up with too few selections.

Better algorithm:

1. Rank all stocks
2. Iterate through list
3. Add stock if sector_count < sectorCap
4. Continue until portfolio_size reached

Example:

sectorCap = 2
portfolioSize = 10

Final portfolio might look like:

Tech: 2
Healthcare: 2
Consumer: 2
Financial: 2
Industrial: 2
4️⃣ Risk-Adjusted Weighting (Your Best Feature)

Normalized weights reduce risk concentration.
