Yes — for what you’re building, I’d keep one core framework but separate the scoring layers modularly.

That gives you the best mix of:

consistency

flexibility

cleaner product design

easier selective access for different audiences

My recommendation

Build it like this:

1. One master framework

This is your common engine:

data ingestion

factor normalization

weighting logic

score calculation

ranking

portfolio rule layer

reporting

Think of this as the operating system.

2. Separate scoring models inside it

Then create separate model packs:

Core Quality model

Opportunity / Mispricing model

Growth model

Fortress model

Wave / momentum-tactical model

Country-specific model variants

maybe later FIC / long-term compounder model vs trading model

Think of these as strategies running on the same OS.

That is usually much better than one giant blended score.

Build into existing framework vs separate scoring
Best path: integrated framework, separate scoring modules
Pros

one data pipeline

one codebase to maintain

easier to compare models

easier to add features later

easier to serve selective audiences

cleaner backtesting because models can share the same historical dataset

avoids duplicated work
