1. You Are Missing a Factor Computation Layer

Right now your design jumps from:

FMP raw data → 120 factor scores

But in reality it must be:

FMP raw financials
      ↓
Derived financial metrics
      ↓
120 factor scores

Example.

FMP gives:

revenue
grossProfit
operatingIncome
totalAssets
cash
debt

But your factors require derived metrics like:

ROIC
incremental margin
cash conversion cycle
FCF yield
R&D intensity
You need a module
/lib/factors/derived-metrics.ts

Example:

export function computeDerivedMetrics(financials) {
  const roic =
    financials.nopat /
    (financials.totalDebt + financials.totalEquity - financials.cash)

  const fcfYield =
    financials.freeCashFlow / financials.marketCap

  const rdIntensity =
    financials.researchAndDevelopmentExpenses /
    financials.revenue

  return {
    roic,
    fcfYield,
    rdIntensity
  }
}

Without this layer the 120-factor model cannot work.

8. Missing Factor Normalization

Different factors operate on different scales.

Example:

ROIC = 0–50
Debt/Equity = 0–5
RSI = 0–100

Before combining scores they must be normalized.

Example:

z-score normalization
or
percentile ranking

Add module:

lib/factors/normalizer.ts

Example:

export function percentileRank(value, dataset){
   const sorted=[...dataset].sort()
   const rank=sorted.indexOf(value)
   return rank/sorted.length
}

Without this the engine weights will be wrong.
